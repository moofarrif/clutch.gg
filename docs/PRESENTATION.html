Core Features

Explorar Partidos

GPS + Haversine para encontrar partidos cerca. Filtro "Todos" o "Cerca de ti". Infinite scroll. Autocomplete de canchas.

Crear Partido

Seleccionar cancha (carrusel con fotos), fecha/hora con picker nativo iOS. Validacion: minimo 5h de anticipacion. Config remoto.

Auto-Draft

10 jugadores se unen, el sistema sortea equipos balanceados por MMR con algoritmo serpentine. Push notification + Socket.io real-time.

Ranking ELO

Sistema de MMR con K-factor dinamico (32 nuevos, 16 veteranos). 5 tiers: Bronce a Diamante. Leaderboard global.

Votacion de Resultado

Post-partido: cada jugador vota ganador. Threshold 60% (6/10). Auto-resuelve y dispara calculo ELO.

Conducta

Califica companeros 1-5 estrellas. Score de conducta visible en perfil. Reputacion como diferenciador social.

Amigos + Escuadra

Busqueda de usuarios, solicitudes de amistad. Squads de hasta 5 con roles (Capitan, Miembro). MMR promedio del squad.

Avatar + Cloudinary

Upload de foto con compresion client-side (800px, WebP, 70%). Cloudinary CDN en produccion, local storage en dev.

Match Status Machine

OPEN → FULL

El 10mo jugador se une. match-join.service detecta count === maxPlayers y cambia status.

FULL → DRAFTING → PLAYING

BullMQ job se encola. Processor ejecuta draft serpentine. Asigna teams A/B. Emite draftComplete via Socket.io.

PLAYING → VOTING → COMPLETED

Jugadores votan resultado. Al alcanzar 60% threshold, el resultado se confirma. ELO job se encola y calcula nuevo MMR.

Formula ELO

K-factor nuevo (<20 matches): 32
K-factor veterano: 16
Expected = 1 / (1 + 10^((opp - mmr) / 400))
New MMR = mmr + K * (actual - expected)
MMR inicial: 1000
Varianza inicial: ±20%

Tech Stack

Frontend
React Native + Expo 54
Expo Router v4, React 19, TypeScript strict, Reanimated v4
State
Zustand + React Query v5
Auth store + server state. Infinite scroll, cache strategy por dato.
Real-time
Socket.io
Match rooms. Events: playerJoined, draftComplete, matchResult, statusChanged.
Backend
NestJS 11
18 modulos. Auth JWT, Passport, guards, Swagger. TypeScript.
Database
PostgreSQL 16 + Drizzle ORM
12 tablas, 22 indexes, Haversine geo queries. Migrations versionadas.
Queue
BullMQ + Redis
2 queues: matchmaking (draft) + elo (calculo). Retry con backoff.
Storage
Cloudinary
Adapter pattern: Cloudinary prod, local dev. Auto WebP, CDN global.
Monorepo
Turborepo + pnpm
3 workspaces: apps/api, apps/mobile, packages/shared. Shared types.
Security
Helmet + JWT Rotation
Access 15min + Refresh 7d en Redis. CORS, rate limiting, WCAG AA.
